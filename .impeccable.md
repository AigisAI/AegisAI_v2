## Design Context

### Users
AegisAI MVP is for developers and security-conscious engineering teams who want to connect GitHub or GitLab repositories and quickly start Java vulnerability scans. They arrive at the login screen as the first trust checkpoint before granting repository access and expect the interface to feel credible, clear, and controlled rather than flashy.

### Brand Personality
Trustworthy, technical, composed. The product should feel like a calm security workspace that helps users move from authentication to repository connection and queued scanning without friction. The interface should communicate seriousness and clarity, while still feeling modern and polished.

### Aesthetic Direction
Use a trust-first editorial security direction. Favor warm neutrals, deep ink tones, and a restrained metallic accent instead of generic dark cyber visuals, neon highlights, or purple AI gradients. The login screen should feel premium and intentional, with asymmetrical layout, strong typographic hierarchy, and clear OAuth action blocks for GitHub and GitLab.

### Design Principles
1. Establish trust before action by making repository access, session-based auth, and Java-first scanning feel explicit and understandable.
2. Prefer calm editorial contrast over loud security clichés, using deliberate typography and restrained color accents.
3. Make the provider choice obvious and low-friction, with clear primary actions and concise support copy.
4. Keep the first screen focused on authentication and forward motion rather than broad product explanation.
5. Preserve mobile clarity and accessibility, especially around action hierarchy, loading states, and error recovery.
